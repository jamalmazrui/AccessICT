WCAG 2.1 Core Accessibility  


17 Items – 260 points available. The team gets full points and 5 points are deducted for each error on each (except #10, #11 and #17 where no points are awarded if any error). Points
1.1 1. Informative Images (20 points) Use the alternative text attribute to provide a short text alternative for non-linked (non-functional) images. If the image is decorative or redundant, use an empty alt attribute (alt=“”) or integrate the image through CSS instead. If the image conveys information, then the image has to be integrated through HTML and the alternative text must convey the intended meaning of the image. If this is not possible with, for example, a chart or graph, then identify the intended meaning of the image with the alternative text attribute and provide the additional information through an in-line description in the content, a link to a text description, or ARIA-describedby. If the image consists of text, the alternative text must contain the text in the image. Each image without appropriate alternative text or lacking an alt attribute is an error.
Teams start with 20 points. Deduct 5 points for each error. If 4 errors found, score is zero for this check.


Errors:

1.1 2. Functional Images (20 points): Images that initiate an action rather than conveying information must have alternative text that relays the purpose of the image. Use the alternative text attribute to convey the function of the image. Use the alternative text attribute to define the function of linked images, image map hot spots, or input elements with type="image". If the image contains text, then the text must be conveyed in the alternative text for the image. In all cases, the text equivalent, whether in the alternative text or as text in an image, must convey the function of the image. Functional images with no or empty alt attributes are counted as one error each. Teams start with 20 points. Deduct 5 points for each error. If 4 errors found, score is zero for this check.


Errors:

2.4.4, 2.4.9 3. Text Links (20 points). Use concise descriptive active elements such as links, menu items, etc. to ensure that each active text element makes sense when it is read out of its immediate context. Thus do not use link text such as "read more", "more", "click here", etc. Each active element with inadequate text is an error.
Teams start with 20 points. Deduct 5 points for each error. If 4 errors found, score is zero for this check.


Errors:

1.3.1,2.4.6, 2.4.10, 4.1.1 4. Correct Markup/Parsing (20 points). Use semantic markup (block quotes, headings, lists, etc.) to properly represent the structure of the document. All elements should have complete start and end tags, be nested according to their specifications, and not contain duplicate attributes. In addition, any IDs are unique, except where the specifications allow otherwise. Each instance of a structural element used for formatting or content that should use semantic markup or that is not structured to specifications is an error.
Do not use table structure to control page layout.

Teams start with 20 points. Deduct 5 points for each error. If 4 errors found, score is zero for this check.


Errors:

2.4.1 5. Skip links (10 points). Unless there are fewer than 4 links between the top of the page and the start of the main content there must be a “skip to main content” that points to the Main landmark. This link must always be visible or must become visible when it receives focus. Using a skip navigation widget is also permissible provided all options in the widget point to a valid ARIA landmark. The link or widget must be keyboard accessible. The "skip to main content" link MUST be the first tab stop on the page.
Teams start with 10 points. Deduct 5 points for each error. If 2 errors found, score is zero for this check.


Errors:

1.3.1, 2.4.1 6. Landmark roles (10 points). Use landmark roles to identify regions. All content on the page must be within an appropriate landmark. Each page that does not meet these requirements is an error. Use aria-label or aria-labeledby to provide a means of distinguishing between content areas that use the same landmark role.
Teams start with 10 points. Deduct 5 points for each error. If 2 errors found, score is zero for this check.


Errors:

1.3.1, 2.4.6, 2.4.10 7. Headings (10 points). Use headings to identify sections and subsections of content. Within the landmark region you must have consistent heading levels. Each page must have one and ONLY one H1. Do not skip heading levels (H2 then H4 without H3 in between would be an error). It is acceptable for heading levels to skip from lower to higher, however (H3, H4, then back to H2 for example would not be an error). Each incorrect use of headings is an error.
Teams start with 10 points. Deduct 5 points for each error. If 2 errors found, score is zero for this check.


Errors:

1.4.1 8. Information in presentation layer (20 points).
Ensure that information conveyed through presentation (such as location, font, color, or font attributes such as bold or italics) is done so with the correct structure. Make sure color alone is not used to convey content or to identify visual elements. Make sure that links in body text are underlined.
Teams start with 20 points. Deduct 5 points for each error. If 4 errors found, score is zero for this check.


Errors:

1.4.3 9. Text Contrast (20 Points). The visual presentation of text and images of text has a contrast ratio of at least 4.5:1 for regular sized text and 3.0:1 for large text. Text is considered "large text" when it is sized at 18 point (roughly 18.66 px) or 14 point bold (roughly 24 px), or larger (read more about large text at https://www.w3.org/TR/UNDERSTANDING-WCAG20/visual-audio-contrast-contrast.html#larger-scaledef). Graphics and user input components shall contrast at a ratio of at least 3:1 against adjacent colors. Contrast shall meet these ratios except for: text or images of text that are part of an inactive user interface component, images that are purely decorative, images that are not visible to anyone (such as a skip to main content link, at least until it becomes visible), or colors in an image that is part of a logo. More details on the Knowbility blog at https://knowbility.org/blog/2018/WCAG21-1411Contrast.
Teams start with 20 points. Deduct 5 points for each error. If 4 errors found, score is zero for this check.


Errors:

2.1 10. Keyboard (20 Points). All interactive elements in the page must be input device independent, specifically, each interactive element must be fully reachable and usable using only the keyboard. All interactive elements should receive keyboard focus order in the same order that they are encountered logically. Keyboard functionality must not require the use of mouse keys, the JAWS cursor, or other similar techniques.
Teams start with 20 points. Any instance of an operation or a control that does not fully function using only the keyboard results in a score of zero for this check.


Errors:

3.1.1 11. Language (10 Points). The correct language of the page is identified in the HTML element, using the lang attribute with a two-character code value set to the current language of the page ("en" for English, for example).
Teams start with 10 points. Any failure to identify the natural language in a page results in a score of zero for this check.


Errors:

1.3.1, 4.1.3 12. Forms (20 points).  For native html form controls with a single label use the label element. For non-native controls use aria-labelledby for visible labels. Use fieldset/legend to group radio buttons and check boxes. Provide a clear visual cue when fields require a user input and use the HTML5 required attribute for required fields and/or the aria-required property. Provide clear instructions to users. This includes expected data formats and relevant information in addition to explanations required to guide the user through the form. Placeholder text is not acceptable as a label or for instructions. See https://www.w3.org/WAI/tutorials/forms/instructions/ for more. Use ARIA markup for error handling including the aria-invalid state on fields which have errors. Use the alert or alertdialog roles for announcing the errors. Code status messages to be recognizable by assistive technology.
Teams start with 20 points. Deduct 5 points for each error. If 4 errors found, score is zero for this check.


Errors:

2.5.3 13. Label in Name (10 points) - For each user interface component that includes a visible text label, the accessible name must match the visible text in the label. Teams start with 10 points. Deduct 5 points for each error. If 2 errors found, score is zero for this check.


Errors:

1.3.5 14. Input Purpose (10 points) Use autocomplete on fields that collect personal data to explicitly identify the data type. Use the W3C's list of Input Purposes for User Interface Components (https://www.w3.org/TR/html52/sec-forms.html#sec-autofill) to select the correct value. Teams start with 10 points. Deduct 5 points for each error. If 2 errors found, score is zero for this check.


Errors:

2.4.7 15. Focus indication (20 points). There must be a clear visual indication when an object receives focus. Focus indication must have color contrast of at least 3 to 1 against the background color adjacent to the focus indicator. Each instance of an object with insufficient focus indication is an error.
Teams start with 20 points. Deduct 5 points for each error. If 4 errors found, score is zero for this check.


Errors:

1.4.4, 1.4.10 16. Text Sizing and Reflow (20 Points) All content can be enlarged (with browser zoom) and responds coherantly to various screen sizes. Text reflows properly when browser page zoom is 400% for all content. Content is not truncated and does not require horizontal scrolling to read. Test by setting the browser viewport to 320 CSS pixels wide by 480 CSS pixels high and browser zoom to 100%, and make sure text reflows and can be read without horizontal scrolling.
Teams start with 20 points. Any instance where either the text cannot be resized or where the text fails to reflow properly when resized is an error


Errors:

1.4.12 17. Text Spacing (10 points) The spacing between letters, words, lines of text and paragraphs can be adjusted by the user. Test using Steve Faulkner's Text Spacing Bookmarklet at zoom level 100% with window size of 1280 wide x 800 high. Exceptions: Video captions, PDF, text implemented in Canvas.
Teams start with 10 points. If any error is found, score is zero for this check.

1.3.4 19. Orientation Adjust content to the orientation of the user device and preference.
Teams start with 5 points. Any violation result is a score of zero for this check. (exceptions if change in orientation fundamentally changes the information or functionality of the content, ie maps)



Errors:

1.4.2, 2.3, 2.3.1, 2.2.2. 20. Distracting Content. Avoid distracting content such as sound, moving text or blinking or flashing areas. Avoid autoplay with multimedia, carousels, and sliders. Ensure webpages do not contain anything that flashes more than three times in any one second. Any animation, moving content, or sounds must stop automatically in 5 seconds or must provide a user controllable method to end the movement or sound.
Teams start with 5 points. Any violation result is a score of zero for this check.


Errors:

3.2.1, 3.2.2 21. Form Controls. All standard HTML native form controls function as they would normally be expected to.
Teams start with 5 points. Any standard HTML native form control that does not function as expected results in a score of zero for this check.


Errors:

4.1.2 22. Links. All hyperlinks function properly.
Teams start with 5 points. Any link not functioning properly (e.g. wrong page, page not found, etc.), results in a score of zero for this check.


Errors:

2.2.1 23. Timing. Except for real-time events where the time limit is absolutely required, if a page or application has a time limit, the user is given options to turn off, adjust, or extend that time limit as often as needed to complete the task.
Teams start with 5 points. Any failure to provide such flexibility results in a score of zero for this check.


Errors:

3.2, 3.21, 3.22, 3.2.5 24. Pop-ups. If content opens in a new browser window or a new tab the user is informed that the content opens in a new window or tab and a method is provided for closing the window or tab. Teams start with 5 points. Any failure yields score of zero for this check.


Errors:

2.4.2 25. Unique Titles. All pages and frames have unique titles that properly describe their topic or purpose.
Teams start with 5 points. Any page or frame title that is not appropriately described results in a score of zero for this check.


Errors:

2.4.5 26. Ease of Navigation. Users can access all significant pages of the site using a well-structured site map or the main navigation menu.
Teams start with 5 points. If no site map or it's equivalent is provided, score is zero for this check.


Errors:

2.4.5 27. Search. An accessible search engine is available, and easy to find. Must have a visible label that is also available to screen reading software, and role = search for points.
Teams start with 5 points. No search or inaccessible search - score is zero for this check.


Errors:

3.1.2 28. Language Changes. The human language of each passage or phrase in the content can be programmatically determined, except for proper names, technical terms, words of indeterminate language, and words or phrases that have become part of the vernacular of the immediately surrounding text. Identify language changes within a page.
Teams start with 5 points. Any failure is score of zero for this check.

1.3.1 35. Data Tables (10 points). Use the CAPTION element and/or the SUMMARY attribute to describe the structure of complex data tables. Use the TH element to unambiguously identify all row and column headers. For complex data tables, associate data cells with the appropriate headers using either HEADERS/ID combination or the SCOPE attribute. Each instance of missing or incorrect table markup is an error and results in a loss of 5 points.
A Team ONLY starts with 10 points IF they indicated they attempted Data Tables in the advanced features of their submission form. Deduct 5 points for each error. If 2 errors found, score is zero for this check. If NO ATTEMPT, score is zero for this check.


Errors:

1.2 36. Video (20 points). For video with soundtrack, provide synchronized captions. Provide an html text description for video without sound. In addition, provide synchronized audio description if the video cannot by understood from the soundtrack alone. If not synchronized, audio description must be provided as text with the video link. Each instance of inadequate accommodation is an error.
A Team ONLY starts with 20 points IF they indicated they attempted VIDEO in the advanced features of their submission form. Deduct 5 points for each error. If 4 errors found, score is zero for this check. If NO ATTEMPT, score is zero for this check.


Errors:

1.2 37. Audio (10 points). Provide html text transcripts for audio files. Speakers must be identified. Each place where the text transcript substantially differs from the audio is an error.
A Team ONLY starts with 10 points IF they indicated they attempted AUDIO in the advanced features of their submission form. Deduct 5 points for each error. If 2 errors found, score is zero for this check. If NO ATTEMPT, score is zero for this check.


Errors:


38. Responsive Design (20 points)
Site is implemented using responsive design techniques to facilitate appropriate presentation on desktop/laptop, tablet and mobile devices. To receive points must support at least 3 device types e.g. smart phone, tablet, and laptop/desktop. All functionality present for smartphone and tablet presentations must work with touch screen interface.
A Team ONLY starts with 20 points IF they indicated they attempted Responsive Design in the advanced features of their submission form. Deduct 5 points for each error. If 4 errors found, score is zero for this check. If NO ATTEMPT, score is zero for this check.


Errors:

1.4.11 39. Non-Text Contrast: (20 points) - Buttons, icons, active user interface components (like text fields, checkboxes and radio buttons), and other graphics required for user to understand or interaction with content have a contrast ratio of 3 to 1 against adjacent color(s).
Judge will consider only if team indicated they attempted "Non-Text Contrast" in the advanced features of submission form. Deduct 5 points for each error. If 4 errors found, score is zero for this check. If NO ATTEMPT, score is zero for this check.


Errors:

1.4.13 40. Content on Hover or Focus (10 points) If dynamic content is added, ensure all 4 of the following are true:
1. Users can move the pointer over additional content to keep it visible
2. Content stays open as long as it is relevant.
3. Content stay open as long as the user is interacting with it.
4. There is a way for the user to dismiss the content.
Out of Scope: Modal dialogues or Tool-Tips.
Judge will consider only if team indicated they attempted "Content on Hover or Focus" in the advanced features of submission form. if attempt is successful, team receives 10 bonus points. If a failure is found or if NO ATTEMPT is indicated, score is zero for this check.
